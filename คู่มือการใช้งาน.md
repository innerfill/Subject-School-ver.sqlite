# คู่มือการใช้งาน ระบบจัดตารางเรียน

---

## การติดตั้ง (ครั้งแรกครั้งเดียว)

### สิ่งที่ต้องเตรียม
- Windows 10 หรือ 11
- ไฟล์ folder โปรแกรมทั้งหมด
- การเชื่อมต่ออินเทอร์เน็ต (สำหรับดาวน์โหลด Node.js)

### ขั้นตอน

**1. ติดตั้งโปรแกรม**
- ดับเบิลคลิก `install.bat`
- รอจนติดตั้งเสร็จ (ใช้เวลา 5–10 นาที ขึ้นอยู่กับ internet)
- ระบบจะติดตั้ง Node.js และ dependencies ให้อัตโนมัติ

**2. เริ่มใช้งาน**
- ดับเบิลคลิก `start.bat`
- รอ build เสร็จ (ครั้งแรกอาจช้าหน่อย)
- เปิด browser ไปที่ `http://localhost:3000`

> ฐานข้อมูล (`school_schedule.db`) จะถูกสร้างอัตโนมัติตอนเปิดใช้ครั้งแรก ไม่ต้องติดตั้ง MySQL หรือ XAMPP

---

## การเปิดใช้งานทุกครั้ง

1. ดับเบิลคลิก `start.bat`
2. รอข้อความ `Open browser at: http://localhost:3000`
3. เปิด browser → `http://localhost:3000`

> ห้ามปิดหน้าต่างดำ (Command Prompt) ขณะใช้งาน — ถ้าปิดระบบจะหยุดทำงาน

## หน้าจอหลัก

### ภาพรวมระบบ (หน้าแรก)
แสดงสถิติภาพรวม:
- จำนวนครู / ห้องเรียน / คาบที่จัดแล้ว
- ความคืบหน้าการจัดตารางแต่ละห้อง

## เมนูหลัก

### ข้อมูลพื้นฐาน
ต้องกรอกข้อมูลนี้ก่อนจัดตาราง **ตามลำดับ**:

| ลำดับ | เมนู | ใช้สำหรับ |
|-------|------|----------|
| 1 | ปีการศึกษา | กำหนดเทอมที่ใช้งาน (ต้องมี 1 เทอมที่ Active) |
| 2 | ข้อมูลทั่วไป | ระดับชั้น / แผนก / อาคาร |
| 3 | ข้อมูลครู | เพิ่ม/แก้ไขข้อมูลครู พร้อมกำหนดสีประจำตัว |
| 4 | รายวิชา | เพิ่มวิชา กำหนดจำนวนคาบ/สัปดาห์ |
| 5 | ห้องเรียน | เพิ่มห้องและความจุ |
| 6 | ชั้นเรียน | เพิ่มห้องเรียน เช่น ม.1/1 |
| 7 | คาบเรียน | กำหนดเวลาแต่ละคาบ |

### จัดการการสอน

| เมนู | ใช้สำหรับ |
|------|----------|
| มอบหมายงานสอน | กำหนดว่าครูคนไหนสอนวิชาอะไร ห้องไหน |
| ล็อกตาราง/กิจกรรม | ล็อกคาบกิจกรรมที่ตายตัว เช่น ลูกเสือ ชุมนุม |
| ภาระงานสอนครู | ดูจำนวนคาบสอนของครูแต่ละคน |
| ตรวจสอบโครงสร้างเวลาเรียน | ตรวจชั่วโมงเรียนตามหลักสูตร |
| ตรวจตารางชน | ตรวจหาปัญหาในตาราง เช่น ครูสอนซ้อน วิชาซ้ำ |

### จัดตาราง
หน้าหลักสำหรับจัดตารางเรียน:
- **ลากวาง** วิชาจาก sidebar ซ้ายไปวางในตาราง
- **ลากย้าย** คาบที่จัดแล้วไปช่องอื่น
- คาบที่มีกุญแจ 🔒 = ล็อกแก้ไขไม่ได้
- ระบบแจ้งเตือนทันทีถ้าตารางชนกัน

### ออกรายงาน
- **ตารางเรียนรายห้อง** — พิมพ์ตารางให้นักเรียน
- **ตารางสอนรายครู** — พิมพ์ตารางให้ครู
- **โครงสร้างรายวิชา** — รายงานวิชาและครูผู้สอน

## ตรวจตารางชน

ระบบตรวจสอบ 4 ประเภท:

| ประเภท | ความหมาย |
|--------|----------|
| โอเวอร์โหลดรายสัปดาห์ | ครูสอนเกิน 25 คาบ/สัปดาห์ |
| โอเวอร์โหลดรายวัน | ครูสอนเกิน 6 คาบ/วัน |
| สอนต่อเนื่องเกิน | ครูสอนติดกันเกิน 3 คาบ |
| วิชาซ้ำในวันเดียว | วิชาเดิมอยู่ในตารางวันเดียวกันมากกว่า 1 คาบ |

## การสำรองข้อมูล

แนะนำ backup ข้อมูลทุกสัปดาห์:
- เข้าเมนู **ตั้งค่า** → แท็บ Backup
- กด **สำรองข้อมูล** → ระบบจะดาวน์โหลดไฟล์ `.db` ให้เก็บไว้

### กู้คืนข้อมูล (Restore)
- เข้าเมนู **ตั้งค่า** → แท็บ Backup
- กด **กู้คืนข้อมูล** → เลือกไฟล์ `.db` ที่สำรองไว้

---

## ปัญหาที่พบบ่อย

| อาการ | วิธีแก้ |
|-------|--------|
| ดับเบิลคลิก `start.bat` แล้วไม่มีอะไรขึ้น | รัน `install.bat` ก่อน — Node.js อาจยังไม่ได้ติดตั้ง |
| เข้า `localhost:3000` ไม่ได้ | ตรวจว่าหน้าต่างดำ (start.bat) ยังเปิดอยู่และแสดงข้อความ `Open browser at:` |
| Build failed ตอนเปิด start.bat | ปิดหน้าต่างดำทุกอัน แล้วเปิด start.bat ใหม่ |
| ข้อมูลหาย | Restore จากไฟล์ `.db` ที่ backup ไว้ ผ่านเมนู ตั้งค่า → Backup |
| ตารางชนแต่ลากวางได้ | ระบบป้องกันการชนอัตโนมัติ — ถ้าชนจะแจ้งเตือนและไม่บันทึก |
| ไฟล์ฐานข้อมูลหาย (`school_schedule.db`) | เปิด start.bat ใหม่ — ระบบสร้างไฟล์ว่างให้อัตโนมัติ จากนั้น Restore จาก backup |
